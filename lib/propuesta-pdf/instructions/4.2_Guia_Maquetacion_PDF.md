# Guía de Maquetación y Exportación a PDF

> Este archivo es para uso exclusivo en Cursor. No forma parte del flujo activo de Notion.
> Se usa una vez que el PM completa la Revisión Final en `Fase3_Propuesta/3.2_Calculos_Automaticos.md` y aprueba el contenido.
>
> Flujo de uso:
> 1. El PM aprueba el borrador completo en Notion (Fase 3 — Revisión Final).
> 2. El PM abre Cursor y copia `templates/PropuestaActualizacion.html` como base.
> 3. El PM sigue esta guía para reemplazar los marcadores `[[...]]` y las filas de tabla con el contenido aprobado.
> 4. Una vez completado el HTML, generar el PDF desde Chrome con los ajustes indicados en esta guía.
>
> Archivos de la plantilla (todos en este repositorio):
> - HTML base: `templates/PropuestaActualizacion.html`
> - Estilos: `templates/PropuestaActualizacion.css`
> - Assets: `templates/assets/`
> - Propuestas generadas: `propuestas_generadas/`

---

## Principio rector

El documento tiene exactamente 5 tablas con campos (columnas) fijos. Nunca se modifican las columnas. Solo se reemplazan las filas y los marcadores `[[...]]` con el contenido aprobado por Notion AI.

| Tabla | Sección en 3.1 | Columnas fijas | Clase CSS |
|---|---|---|---|
| Soluciones | Sección 3 | Módulo / Complejidad / Descripción / Funcionalidades principales | `data-table-solutions` |
| Personal | Sección 6 | Rol / Cantidad / Descripción del Rol | `data-table-personal` |
| Actividades | Sección 7 | Actividad / Descripción / Tiempo estimado (semanas) / Tiempo estimado (horas) | `data-table-activities` |
| Requerimientos y Tiempos | Sección 8 | # / Requerimiento / Descripción / Tiempo | `data-table-requirements` |
| Tiempos y Costos — Etapas | Sección 9.1 | Etapas / Tiempo estimado / Detalle | `data-table-stages` |
| Tiempos y Costos — Precios | Sección 9.2 | Descripción / Precio | `price-table` |

## Prompt — Generador de Forma de Pago para el PDF

> Ejecutar cuando el esquema de pagos esté confirmado en Notion (3.2 — Prompt Esquema de Pago).

> (AI) **PROMPT — Texto de Forma de Pago para el PDF (PM ejecuta)**
>
> Lee las propiedades `Subtotal_USD` y `Esquema_Pagos` de esta página. Consulta también el Histórico de Proyectos para verificar qué hito se usó en proyectos similares.
>
> Genera el texto con exactamente este formato por cada pago:
>
> - Fase [N] — [Nombre del hito]
>   - Se debe pagar el [X] % del pago para iniciar el proyecto con un monto total de $[monto con formato ##,###.00] ([monto en letras] dólares americanos) sin IVA.
>
> Reglas:
> - Los montos se calculan sobre el SUBTOTAL (sin IVA), no sobre el total con IVA.
> - El monto en letras debe estar en español: `mil trescientos noventa y dos` (sin la palabra "con", sin decimales en las letras).
> - Usar coma para miles y punto para decimales: `$1,392.00`.
> - Nombres de hitos estándar:
>   - 1er pago: "Inicio de proyecto"
>   - 2do pago (2 pagos): "Entrega aplicativo en desarrollo"
>   - 2do pago (3 pagos): "Entrega acta de pruebas unitarias"
>   - 3er pago (3 pagos): "Entrega aplicativo en desarrollo"
>
> Luego pregunta: ¿Los nombres de los hitos son correctos para este cliente?
>
> Al finalizar, responde obligatoriamente:
> 1. ¿Dónde en Notion guardo este texto aprobado?
> 2. Estado de este paso:
>    - [ ] Está bien — texto listo para pegar en el HTML
>    - [ ] Necesita cambios — describir qué ajustar antes de continuar

## Proceso de exportación

**Paso 1 — Preparar el HTML**
- Copiar `templates/PropuestaActualizacion.html` a `propuestas_generadas/[Codigo_Propuesta]_Propuesta_[Cliente].html`.
- Abrir la copia en el editor de código.
- Verificar que la ruta de los assets es `../templates/assets/` (si el archivo vive en `propuestas_generadas/`). Si prefieres mantener la ruta `assets/`, copiar también la carpeta `assets/` dentro de `propuestas_generadas/`.

**Paso 2 — Reemplazar marcadores y filas**

| Sección del HTML | Acción |
|---|---|
| `[[NOMBRE_PROPUESTA]]`, `[[CODIGO_PROPUESTA]]`, `[[VERSION]]`, `[[FECHA_PROPUESTA]]`, `[[VALIDEZ_DIAS]]` | Reemplazar en Portada, Índice y `<title>` |
| `[[SCRUM_MASTER]]`, `[[QA_RESPONSABLE]]` | Roles SCRUM |
| `[[OBJETIVO_N]]` | Objetivos aprobados en Notion |
| `data-table-solutions` → filas | Una fila por módulo aprobado en tabla Soluciones |
| `data-table-personal` → filas | Una fila por rol aprobado en tabla Personal |
| `data-table-activities` → celdas | Reemplazar `[[ACT_N_*]]` en las 9 filas de Actividades |
| `data-table-requirements` → filas | Una fila por requerimiento aprobado en tabla Requerimientos y Tiempos |
| `data-table-stages` → celdas | `[[HORAS_DESARROLLO]]`, `[[HORAS_UAT]]`, `[[TOTAL_HORAS]]` |
| `price-table` → celdas | `[[PRECIO_DESARROLLO]]`, `[[PRECIO_PRUEBAS]]`, `[[SUBTOTAL]]`, `[[IVA]]`, `[[TOTAL]]` |
| `payment-list` | Dejar solo las fases del esquema aprobado y reemplazar `[[PAGO_N_*]]` |

**Paso 3 — Verificar imágenes**
- Confirmar que los 4 archivos de imagen están accesibles desde la ruta usada en el HTML.
- Si al abrir en Chrome alguna imagen no carga, revisar la ruta relativa.

**Paso 4 — Generar el PDF en Chrome**
- Abrir el HTML en Google Chrome.
- Presionar `Ctrl+P` → Guardar como PDF.
- Configuración obligatoria:
  - Tamaño de página: **Carta (Letter)**
  - Márgenes: **Ninguno (None)**
